# Code of Conduct

## Purpose

Phoenix is a technical project, but collaboration still depends on clear and respectful behavior. This code of conduct sets expectations for contributors, maintainers, and anyone participating in the project space.

## Expected behavior

Participants are expected to:

- Communicate respectfully, even during technical disagreement
- Focus feedback on code, design, requirements, and evidence
- Assume good intent unless there is a clear reason not to
- Make room for questions, learning, and different experience levels
- Keep discussion relevant, constructive, and professional

## Unacceptable behavior

The following behavior is not acceptable:

- Personal attacks, insults, harassment, or intimidation
- Discriminatory or demeaning language related to identity or background
- Repeatedly derailing discussions or intentionally wasting other contributors' time
- Publishing private, sensitive, or personal information without permission
- Sexualized language, unwanted advances, or other conduct that makes participation unsafe
- Threats, stalking, or any other abusive behavior

## Scope

This code of conduct applies to project spaces such as issues, pull requests, discussions, code review, documentation, and other collaboration channels maintained for Phoenix.

## Reporting concerns

If you experience or witness behavior that violates this code of conduct, contact the project maintainers through a private channel when available on the hosting platform.

If no private contact path is available, open a minimal issue requesting private follow-up without posting sensitive details publicly.

Include any relevant context, links, screenshots, or timestamps that would help the maintainers assess the report.

## Maintainer responsibilities

Project maintainers are responsible for:

- Reviewing reports fairly and promptly
- Protecting reporter privacy as much as practical
- Taking proportionate action based on the severity and pattern of behavior
- Prioritizing the safety and working ability of the community over argument about intent

Possible responses may include a warning, comment removal, temporary restriction from participation, or removal from project spaces.

## Enforcement

Maintainers may moderate any contribution or interaction that violates this code of conduct. Repeated or serious violations may result in suspension or a permanent ban from participation.

## Project standard

When there is ambiguity, default to professionalism, clarity, and respect.
